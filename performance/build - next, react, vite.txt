🧠 1. MEMORY vs DISK (CORE CONCEPT)

🔹 Disk (Storage)
👉 Your SSD / HDD

    Persistent (stays after restart)
    Slower than memory
    Example:
    .next/
    dist/
    build/

🔹 Memory (RAM)
👉 Temporary working space

    Super fast ⚡
    Cleared when process stops
    Used by dev servers


🧠 2. What is “compiling” in frontend?
    const App = () => <h1>Hello</h1>;

    Browser cannot understand JSX
    So tools convert it to:
        const App = () => React.createElement("h1", null, "Hello");

    👉 This process = compile / transform    

🧠 3. Now comes the BIG difference    
    ❓ Where does this compiled code go?
    Two options:
    Option A → Store in MEMORY
        Fast
        Temporary
    Option B → Store on DISK
        Persistent
        Used in production

🚀 4. Next.js (DETAILED)
    🔹 npm run dev
    👉 What happens:
        Start dev server
        No full build
        Pages compile only when visited
        🧠 Storage breakdown:-

            🧠 Memory (MAIN)-
                Active pages live here
                Served to browser

            📦 .next (CACHE)-
                Stores some compiled files
                Helps avoid recompiling 

            Flow
                Open /home
                → compile
                → store in memory + cache in .next

                Open /dashboard
                → compile
                → store in memory + cache in .next      
            key point - .next != full app, It is just - .next = helper cache

        🔹 npm run build
            👉 What happens:
                Compile EVERYTHING
                Optimize:
                minify JS
                split chunks
                generate CSS            
                .next = FULL APPLICATION

        🔹 npm run start
                Reads ONLY from .next
                No compiling
                No memory tricks    

        ⚡ NEXT.JS SUMMARY

        DEV:
            memory = main
            .next = cache

        PROD:
            .next = everything
            memory = just serving

⚡ 5. VITE (VERY IMPORTANT – MODERN APPROACH)
    🔹 npm run dev
        - No bundling
        Browser requests file → Vite transforms → sends it
        Uses native ES modules
        Only sends what browser asks
        Very fast

    🧠 How it works
        Browser does this:
            import App from './App.jsx'
            Vite:
                Converts JSX → JS
                Sends file        
                👉 Only when requested

        🧠 Storage
        🧠 Memory
            Transformed modules cached        
        📦 Disk   
            node_modules/.vite 
            vite uses ES Modules(it is available in vite itself) to pre-bundles node_modules and cache it

React (CRA - npm start)
    👉 Full bundling (in memory)

    Entire app → bundled → stored in memory → served

    React uses - babel for transpile and webpack for bundle


Note - All CRA vite next js - bundle node modules
                ----XXXX----

Summary of all-
    ⚛️ React (CRA)
        Transpile → Bundle everything → Store in memory → Serve
        Uses: Babel + Webpack
        node_modules: ✅ bundled
        Memory: ✅ main
        Disk: ❌ almost unused

    🚀 Next.js
        Transpile → Bundle per page (on demand) → Store in memory → Cache in .next → Serve
        Uses: SWC to transpile (Replacement for Babel)
        node_modules: ✅ bundled (per page)
        Memory: ✅ main
        Disk (.next): ✅ cache (partial, unstable)    

    ⚡ Vite
        Dependencies → pre-bundled once → cached in node_modules/.vite

        Your code:
        Browser requests → Transpile → Serve (no bundling)   
        Uses: ESBuild
        node_modules: ✅ pre-bundled separately
        Your code: ❌ not bundled
        Memory: ✅ main
        Disk: .vite cache 

    🏭 PRODUCTION MODE (ALL TOOLS SAME IDEA)

        Everything → Transpile → Bundle → Optimize → Store on disk → Serve

    🧠 ONE-LINE DIFFERENCE
        CRA = bundle everything
        Next.js = bundle smartly (per page)
        Vite = don’t bundle (in dev), just serve modules    


❓ What happens in npm run build?
    CRA → Babel + Webpack bundle
    Vite → esbuild + Rollup bundle
