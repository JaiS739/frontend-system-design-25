Whenever content on the screen changes (like a new message, alert, or status),
aria-live determines when and how urgently screen readers should announce it.



| Value       | Meaning                                                     | Example Use                     |
| ----------- | ----------------------------------------------------------- | ------------------------------- |
| `off`       | Screen reader ignores changes (default)                     | Non-important updates           |
| `polite`    | Screen reader waits until user finishes current speech      | Notifications, status updates   |
| `assertive` | Screen reader interrupts immediately to announce the change | Error messages, critical alerts |
