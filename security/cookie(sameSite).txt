So when we should set the cookie from backend only for access token and refresh token
here are the properties fro cookies setting from backend 
    - SameSite
    - Secure
    - HttpOnly


there are three types of SameSite
    - None - Only same-site requests Requires Secure=true
    - Lax - Send cookie in normal cases, but not in risky cross-site requests
    - Strict - Only same-site requests

Secure works with HTTPS
    so if we are using localhost which is generally http so we can not set secure as true

    so for local 
        SameSite as Lax
        secure as false
    but for production is should be 
        SameSite as none
        Secure as true
