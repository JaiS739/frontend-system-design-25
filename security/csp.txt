🧠 What is CSP?

CSP (Content Security Policy) is a powerful security feature built into browsers that helps protect your website from:

Cross-Site Scripting (XSS)

Clickjacking

Data injection attacks

Mixed content (HTTP inside HTTPS)

You set CSP using an HTTP response header that tells the browser:

“What sources of content are allowed to load on my site.”

👉 It is a rulebook for the browser
    It tells the browser:
    “From where am I allowed to load things?”


🎯 Why CSP exists
    When a user opens your website:
👉 The browser loads:
    JavaScript
    CSS
    Images
    Fonts
    API data

    Now imagine a hacker injects this:    
        <script>stealCookies()</script>
        💥 Without CSP → browser runs it
        ✅ With CSP → browser blocks it

    | Type          | Meaning       | Example                       |
    | ------------- | ------------- | ----------------------------- |
    | `script-src`  | JavaScript    | React app, analytics          |
    | `style-src`   | CSS           | Ant Design styles             |
    | `img-src`     | Images        | logos, avatars                |
    | `connect-src` | API calls     | fetch / axios                 |
    | `font-src`    | Fonts         | Google Fonts                  |
    | `default-src` | fallback rule | applies if others not defined |
        

🔍 Super Simple Analogy

    Think your app is a house 🏠
    CSP is the security guard at the gate
👉 CSP says:
    “Only these people are allowed inside”     


❓ Do I need CSP in frontend OR backend?       

    👉 Short answer:
        ✅ You only need to send CSP from ONE place
        👉 usually backend

🧠 Why backend is preferred
    Because:
        Backend sends HTTP headers
        Browser trusts headers more
        Cannot be modified easily
